# feat(feedback): 实验反馈闭环模块规范建设、日志体系开发与验证

## 背景

超快激光加工工艺决策智能体在推荐加工参数时，依赖**机器学习回归模型**和**历史案例匹配**两种策略生成候选参数。然而，推荐结果的质量高度取决于训练数据的覆盖度与时效性——新材料、新工艺目标的出现会迅速暴露模型的泛化边界。

目前系统存在以下瓶颈：

1. **实验反馈链路断裂**：实验员完成加工后，实测数据（深度、粗糙度）以口头或零散笔记形式流转，无法系统性回流到模型训练管道。
2. **日志格式混乱**：不同操作员、不同批次的实验记录缺乏统一模板，难以自动化解析，更无法批量导入数据集。
3. **反馈数据不可追溯**：一次推荐→实验→实测→回流的完整链路没有唯一标识，出了问题无法定位是推荐偏差还是实验执行偏差。
4. **模型迭代无抓手**：没有持续的反馈样本注入，模型只能依赖静态的历史 Excel 数据，推荐的准确率天花板被锁死。

为解决上述问题，需要建设和规范化**实验反馈闭环**——从推荐生成到实验执行，再到实测回流与模型迭代，形成可量化、可审计、可自动化的完整数据飞轮。

## 目标

### 总体目标
建立一套端到端的实验反馈闭环规范与工具链，使每一次超快激光加工实验的结果都能**结构化地回流**到模型训练数据集，持续提升推荐的准确率与鲁棒性。

### 具体目标

| 序号 | 目标 | 关键成果 |
|------|------|----------|
| 1 | **反馈 API 规范化** | 定义 `ExperimentFeedback` 数据契约（含任务输入、选中参数、实测质量），提供 `POST /api/feedback` 端点 |
| 2 | **双路持久化** | 反馈数据同时写入 JSONL（文本可读、Git 友好）和 SQLite（结构化查询），保证数据不丢失 |
| 3 | **标准化日志生成器** | 开发 `auto-feedback-logger.py`，支持交互模式和 JSON/JSON 文件输入模式，自动生成格式统一的实验反馈日志 |
| 4 | **日志模板规范化** | 定义三段式日志结构：系统推荐参数 → 实测结果与误差分析 → 反馈元信息，作为实验文档的强制标准 |
| 5 | **反馈数据自动注入训练集** | `data_loader.py` 在加载数据集时自动合并 `feedback.jsonl` 中的反馈样本，与新批次数据一起参与模型训练 |
| 6 | **Web 端反馈入口** | 在推荐结果页面提供"追加反馈"组件，实验员可直接选择推荐项并填写实测数据，一键提交 |
| 7 | **端到端验证** | 完成至少 5 种材料（4H碳化硅、BF33、微晶玻璃、金刚石、高温合金）的反馈闭环走通，验证数据从录入到模型生效的完整链路 |

## 闭环流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      实验反馈闭环（Feedback Loop）                        │
└─────────────────────────────────────────────────────────────────────────┘

  ┌──────────┐      ┌──────────────┐      ┌──────────────┐      ┌──────────┐
  │ Step 1   │      │   Step 2     │      │   Step 3     │      │ Step 4   │
  │ 参数推荐  │ ───→ │  实验执行     │ ───→ │  实测数据录入  │ ───→ │ 模型迭代  │
  └──────────┘      └──────────────┘      └──────────────┘      └──────────┘
       ↑                                                            │
       └────────────────────────────────────────────────────────────┘
                              反馈数据回流

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1 — 参数推荐
┌─────────────────────────────────────────────────────────────────────────┐
│ 用户在 Web 端填写任务需求：                                                │
│   材料（material）        如 BF33 / 4H碳化硅 / 微晶玻璃 / 金刚石 / 高温合金  │
│   目标深度（target_depth_um）    如 5.42 μm                                │
│   粗糙度上限（max_roughness_um） 如 0.45 μm                                │
│                                                                         │
│ 系统调用推荐引擎（`apps/api/app/services/recommender.py`）：               │
│   策略 A — 回归模型拟合（ml_regression_fit）                                │
│     使用当前数据集训练的随机森林模型，直接预测最优参数组合                      │
│   策略 B — 历史案例匹配（historical）                                      │
│     基于目标深度/粗糙度，在历史数据中匹配最相似的成功案例                       │
│                                                                         │
│ 输出：Top-K 推荐参数列表，每条附带得分、预测质量、不确定性区间、物理解释        │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
Step 2 — 实验执行
┌─────────────────────────────────────────────────────────────────────────┐
│ 实验员根据推荐结果选择一条推荐（或根据经验调整参数后组合），                      │
│ 在超快激光加工设备上完成加工实验。                                           │
│                                                                         │
│ 关键要求：                                                                │
│   - 记录实际使用的参数（若偏离推荐，需注明修改原因）                            │
│   - 保留加工样品用于后续检测                                                │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
Step 3 — 实测数据录入
┌─────────────────────────────────────────────────────────────────────────┐
│ 实验员对加工样品进行检测，获得实测加工深度和表面粗糙度。数据录入有两条路径：      │
│                                                                         │
│ ┌─ 路径 A：Web 端在线提交 ─────────────────────────────────────────────┐ │
│ │ 在推荐结果页面点击"追加反馈"按钮：                                      │ │
│ │   1. 选择所采用的推荐编号（rank）                                       │ │
│ │   2. 填写实测加工深度（μm）和实测表面粗糙度（μm）                        │ │
│ │   3. 填写实验备注（可选）                                               │ │
│ │   4. 点击提交 → `POST /api/feedback`                                  │ │
│ │                                                                       │ │
│ │ API 端（`apps/api/app/services/feedback.py`）：                        │ │
│ │   - 生成唯一 feedback_id（UUID4）                                     │ │
│ │   - 记录 UTC 时间戳                                                   │ │
│ │   - 双路持久化：                                                       │ │
│ │       ① `data/feedback/feedback.jsonl`（追加一行 JSON）               │ │
│ │       ② `data/feedback/feedback.db`（写入 SQLite 表）                 │ │
│ │   - 返回 FeedbackReceipt（feedback_id, created_at, 存储路径）          │ │
│ └───────────────────────────────────────────────────────────────────────┘ │
│                                                                         │
│ ┌─ 路径 B：离线日志生成器 ──────────────────────────────────────────────┐ │
│ │ 使用 `scripts/auto-feedback-logger.py`：                               │ │
│ │                                                                       │ │
│ │   交互模式：                                                            │ │
│ │     python scripts/auto-feedback-logger.py                            │ │
│ │     → 选择材料 → 输入目标 → 录入推荐参数 → 录入实测结果                     │ │
│ │                                                                       │ │
│ │   JSON 模式（自动化集成）：                                              │ │
│ │     python scripts/auto-feedback-logger.py --json '<JSON>'            │ │
│ │     python scripts/auto-feedback-logger.py --json-file <路径>          │ │
│ │                                                                       │ │
│ │   输出：标准化三段式 txt 日志 → `experiments/实验数据反馈日志/`            │ │
│ │                                                                       │ │
│ │   日志结构（由 `generate_log()` 生成）：                                 │ │
│ │     1. 系统推荐参数 — 推荐方法、得分、各参数值、预测质量                     │ │
│ │     2. 实测结果 — 选用推荐、加工深度、表面粗糙度、误差分析                    │ │
│ │        - 深度偏差（μm / %）                                            │ │
│ │        - 深度绝对误差（μm）                                             │ │
│ │        - 粗糙度余量（μm）                                              │ │
│ │        - 粗糙度达标判定（OK / 超标）                                     │ │
│ │        - 预测深度偏差、预测粗糙度偏差                                     │ │
│ │     3. 反馈录入 — 操作员、生成时间、工具签名、备注                          │ │
│ │                                                                       │ │
│ │   文件命名：`YYYY MM DD 材料名称（N）.txt`（冲突自动递增序号）              │ │
│ └───────────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
Step 4 — 模型迭代（数据回流与再训练）
┌─────────────────────────────────────────────────────────────────────────┐
│ 阶段 A — 数据集自动合并                                                   │
│   每次调用推荐接口时，`data_loader.py::load_dataset()` 自动执行：             │
│     1. 读取 `data/raw/` 下的所有 Excel 文件（原始实验数据）                  │
│     2. 读取 `data/feedback/feedback.jsonl`（反馈样本）                     │
│     3. 统一映射参数列（PARAMETER_COLUMNS）和质量列（QUALITY_COLUMNS）        │
│     4. 标记来源（source_file = "feedback.jsonl" vs Excel 文件名）          │
│     5. 合并为完整 DataFrame，供模型训练使用                                  │
│                                                                         │
│ 阶段 B — 反馈样本统计                                                      │
│   `dataset_summary()` 返回 `feedback_samples` 计数，可在仪表盘监控反馈回流速率│
│                                                                         │
│ 阶段 C — 模型重训练（待建设）                                               │
│   触发条件（满足其一）：                                                    │
│     - 反馈样本数较上次训练时增量 ≥ N 条（如 5 条）                            │
│     - 某材料分类下的反馈样本占比较上次训练时增加 ≥ 20%                          │
│     - 用户手动触发重训练                                                    │
│   重训练完成后：                                                           │
│     - 模型版本号递增（如 0.4.0 → 0.4.1）                                   │
│     - 记录训练数据集快照（原始样本数 + 反馈样本数 + 时间戳）                      │
│     - 后续推荐请求自动使用新模型                                             │
└─────────────────────────────────────────────────────────────────────────┘
```

## 技术实现概要

### 文件结构

```
project_root/
├── apps/api/app/
│   ├── schemas.py              # ExperimentFeedback / FeedbackReceipt 数据模型
│   ├── routers/feedback.py     # POST /api/feedback 路由
│   └── services/
│       ├── feedback.py         # append_feedback(): JSONL + SQLite 双路写入
│       └── data_loader.py      # load_dataset(): 自动合并 feedback.jsonl 到训练集
├── apps/web/components/
│   ├── feedback-form.tsx       # "追加反馈"UI 组件
│   └── workbench.tsx           # 工作台，集成反馈表单
├── scripts/
│   └── auto-feedback-logger.py # 离线日志生成器（交互 + JSON 模式）
├── data/feedback/
│   ├── feedback.jsonl          # 反馈数据（JSONL，文本可读）
│   └── feedback.db             # 反馈数据（SQLite，结构化查询）
└── experiments/实验数据反馈日志/
    └── YYYY MM DD 材料.txt     # 标准化实验日志
```

### 数据契约（ExperimentFeedback）

| 字段 | 类型 | 说明 |
|------|------|------|
| `task` | `RecommendationRequest` | 原始任务输入（材料、目标深度、粗糙度上限） |
| `selected_parameters` | `dict[str, float]` | 实验实际采用的加工参数 |
| `measured_quality` | `dict[str, float]` | 实测质量指标（measured_depth_um, measured_roughness_um） |
| `operator` | `str \| None` | 操作员标识 |
| `notes` | `str \| None` | 实验备注 |

### 材料-参数映射

日志生成器内置了 5 种材料的参数配置：

| 材料 | 核心参数 |
|------|---------|
| 4H碳化硅 | 脉冲宽度、重复频率、扫描速度 |
| BF33 | 脉冲宽度、重复频率、扫描速度 |
| 微晶玻璃 | 脉冲宽度、重复频率、扫描速度、离焦量、扫描间距、加工时间 |
| 金刚石 | 脉冲宽度、重复频率、扫描速度、标记次数、填充间距 |
| 高温合金 | 重复频率、能量档位、脉冲能量、离焦量、标记次数、平均功率 |
